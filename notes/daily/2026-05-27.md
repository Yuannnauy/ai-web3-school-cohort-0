## Core Takeaways

- [ ] **Oracle is not a "real-world API" — it's a data submission and verification mechanism.** Price feed error → liquidation error → user assets lost. Contracts reading feeds must check: update time, decimals, and outlier values.
- [ ] **Indexing turns raw block streams into business-ready query layers.** Users care about "my position," not "what happened in block #123456." RPC reads state; Indexing handles complex historical queries at scale.
- [ ] **Agent's data stack = Oracle + Indexing + RAG.** Oracle gives off-chain world data, Indexing gives on-chain history, RAG gives documentation knowledge. All three together = Agent has complete context for decisions.
