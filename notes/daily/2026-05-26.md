## Core Takeaways

- [ ] **AA turns "has key / no key" into "allowed to do X under conditions Y."** Multi-sig, gas sponsorship, session keys, spending limits, batch transactions — all variants of the same idea.
- [ ] **Session Key = temporary, scoped, revocable permission token for Agents.** Scope it to specific contracts, methods, amounts, durations, chains. Sits between "do nothing" and "here's my private key."
- [ ] **AA + Memory = complete Agent blueprint.** Memory decides what the Agent remembers (who, preferences, history). AA decides what the Agent is allowed to do (how much, how long, what scope).
